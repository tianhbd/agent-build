# acceptance.md (v0.12)

## 全局验收规则

1. 所有变更必须映射 `TASK-####`。
2. 节点类型判定必须可追溯。
3. composite 节点必须提交完整阶段协议与证据。
4. subagent 边界必须合规。
5. 双层质量控制必须通过。
6. reviewer accept 才可 done。

## 双层质量控制验收

### n8n 流程级

1. JSON 结构合法。
2. 必填字段完整。
3. 基础可执行路径有效。

### 节点内容级

1. 逻辑正确。
2. 结构合理。
3. 节奏一致。
4. 阶段推进决策有证据。

## memory 验收

1. 长期写回符合三条件（多次验证、质量提升、reviewer间接确认）。
2. subagent 无长期 memory 写入。
3. Ephemeral 未违规升格。
